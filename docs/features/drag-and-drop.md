# Feature: 拖放（Drag & Drop）

## 一句话

把文件 / 选中文本 / URL 拖到桌宠身上，它会朝你这边凑过来张嘴接住。这是桌宠形态相对于聊天窗口的最大优势。

## 接受的内容类型

| 类型 | 来源举例 | 处理方式 |
|---|---|---|
| **文件** | Finder 拖出的单个或多个文件 | **以 `@<绝对路径>` 形式拼到 `-q` 文本里**，让 Hermes 自己用工具读；多文件用空格分隔（V1 决策，详见 [../tech.md](../tech.md) §11） |
| **选中文本** | 任意应用里选中拖出的文字（Safari、VSCode、Notes 等） | 文本作为输入内容直接拼到 `-q` |
| **URL** | 浏览器地址栏拖出的链接 | URL 字符串作为输入，Hermes 端可调 fetch 类工具 |
| **图片** | Finder 或网页拖出的图片 | 视为文件路径处理（`@<path>`）；后续可以让 Hermes 直接看图（取决于 model 能力） |

## 普通模式下的拖放

1. 拖动开始（鼠标按住任何东西移动）→ 桌宠"察觉"，头朝鼠标方向偏一点
2. 拖到桌宠身上 → 嘴张开，整体微微向鼠标侧倾斜
3. 松开 → 嘴合上 + 一次小弹跳
4. 接住后的内容**默认进入"对话"输入框** —— 但因为现在是普通模式，对话框看不见。**待 confirm**：
   - **方案 A**：自动切到工作模式 + 三气泡浮出 + 对话框已预填 → 用户按回车就发
   - **方案 B**：内容暂存，下次进入工作模式时自动填入对话框
   - **方案 C**：弹一个浮动小提示"已收到，去工作模式发送" → 用户手动切

## 工作模式下的拖放

工作模式下精度更高，因为有三个明确目标。

1. 拖动开始 → 三个气泡自动浮出（即使没 hover）
2. 拖动到具体气泡上方 → 该气泡边框变粗 + 高亮
3. 拖动到桌宠头上但没指定气泡 → 默认对话气泡高亮（暗示松开会进对话）
4. 松开：
   - 在某气泡上 → 内容写入该气泡输入框
   - 在桌宠头上 → 内容进入对话气泡输入框
   - 在桌宠/气泡之外 → 当作未发生，啥也不做
5. 内容进入输入框后**不自动提交** —— 用户检查再回车（避免误发）

## 张嘴接住动画

- 触发：鼠标按住拖动状态进入桌宠包围盒
- 动作：
  - 头部朝鼠标方向偏 ~10°
  - 嘴张开（弧线从 `M ... Q ... ...` 变成稍大的开口）
  - 露出来的翅膀（工作模式）或两只翅膀（普通模式）抖一下
- 离开包围盒（没松开就拖走）：恢复原状
- 松开：嘴合上 + 整体下沉 2px 再回弹（200ms）

## 多文件拖放

- 拖入多个文件 → 全部接住
- 工作模式：批量进入指定气泡的输入框，文件名以列表形式显示在输入区上方（"📎 file1.pdf, file2.txt, ..."）
- 提交时一起作为输入传给 CLI
- 单个气泡处理多附件的能力取决于 Hermes CLI 协议（待定）

## 拒绝的内容

- 拖入超大文件（> 设置里阈值，默认 100MB）→ 弹出小提示"太大了我吃不下"，不接受
- 拖入空选区 → 不响应

## 与其他特性的关系

- 接住后写入的目标气泡 → 对应的 [research](research-bubble.md) / [对话](dialog-bubble.md) / [cowork](cowork-bubble.md) 文档
- 文件路径如何传递给 CLI → 见上"接受的内容类型"，**`@<path>` 拼到 `-q`** 是 V1 决策（详见 [../tech.md](../tech.md) §3 / §11）

## 视觉规格

- 拖动时桌宠倾斜角度：~10° 朝鼠标
- 张嘴时嘴的尺寸放大 ~50%
- 气泡高亮：边框从 0.8px → 2px，颜色保持工作模式蓝 #5DA8E8

## 验收标准

- 拖入响应延迟 < 50ms（任何来源）
- 拖出桌宠包围盒 → 张嘴动作 100ms 内还原
- 多文件批量接住正确处理
- 大文件提示在文件触发拖入但未松开前就显示（在拖到桌宠上但不接受）
