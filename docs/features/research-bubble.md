# Feature: research 气泡

## 一句话

工作模式下三个气泡的最上面一个 —— 给 Hermes 一个"研究主题"，它套着一个万能分析骨架去调研，做完挥手提醒，点开看结果。

## 输入

- placeholder：`研究主题…`
- 单行输入；回车提交
- 也支持拖入：拖动文件 / URL / 文本到该气泡上松开 → 内容写入输入框，等用户回车（或自动提交，**待 confirm**）

## System Prompt（V1）

V1 不做多模板可选 —— **就一个万能分析骨架**，写死在桌宠端，随气泡一起发给 Hermes CLI。

骨架内容（建议草稿）：

```
你是一个调研助手。当我给你一个研究主题或问题时，请按以下框架回复：

1. 拆问题：把这个题目拆成 3-5 个子问题
2. 列假设：你目前的初步判断和不确定的地方分别是什么
3. 找证据：列出你认为最值得查的来源、最关键的事实、最相关的数据点
4. 给结论：在以上基础上给一个有立场、可被反驳的结论

回复保持精简，每个部分用一段话或一个短列表。不要客套，不要免责声明。
```

> 这是骨架草稿。正式版可在桌宠 settings 里改默认值（详见 settings.md）。

## 后端调用

桌宠 spawn 一次性的 Hermes CLI 子进程，**system prompt 拼到 `-q` 文本最前面**：

```bash
hermes chat -Q --accept-hooks \
  -q "<research_system_prompt>\n\n<user_input>"
```

- 单轮、不带 `-r/-c`（research 不保留上下文，每次新 session）
- `-Q` 关 banner / spinner，让 stdout 干净
- `--accept-hooks` 桌宠是无 TTY 环境必须开
- system_prompt 取自 settings.system_prompts.research（详见 [settings.md](settings.md)）

> `--source tool` 原计划用于隔离桌宠会话；2026-04-29 实测与 `-Q` 组合会明显变慢，V1 暂不带。

详见 [../tech.md](../tech.md) §3.2 / §3.3。

## 视觉反馈

- 输入完回车提交 → 工作模式整体进入 `task_running`（红脸）
- stdout 流过来的过程中：**只在后台累积，不实时显示在气泡里**（避免气泡闪烁分心）
- 流结束（子进程退出）→ 红脸消退 + 挥手 + 气泡上出现一个不显眼的小红点 / 数字标记
- 用户点击 research 气泡 → 展开查看结果

## 结果展开形式

点击 research 气泡：

- 当前气泡向下展开成一个浮窗（宽 380px，高自适应，最大 400px 后内滚）
- 浮窗内：
  - 顶部：原研究主题（灰色一行小字）
  - 主体：Hermes 输出（按 Markdown 渲染：列表、代码块、强调）
  - 底部：一个"复制"按钮 + 一个"丢给对话气泡继续聊"按钮
- 点击浮窗外 / 按 Esc → 收起（结果保留，红点清掉）

## 错误情况

- 子进程非 0 退出 / stderr 有内容 → 桌宠头上出"困惑"表情（眼睛变成 `>_<` 或类似），气泡上小红点变成红色
- 点击展开 → 显示错误内容 + 一个"重试"按钮

## 历史记录

V1 不持久化 —— research 每次都是独立 session（不带 `-r`），关掉桌宠或切到普通模式再回来时已经丢失。  
V2 可以做"最近 N 次"，技术上优先靠 Hermes session 能力恢复历史；若 `--source tool` 性能问题解决，再考虑按 source 过滤桌宠会话。

## 验收标准

- 输入回车 → 红脸出现 < 100ms
- 流结束 → 挥手 + 红脸消失 < 500ms
- 点击气泡 → 浮窗展开 < 200ms，内容已渲染好
- 浮窗滚动顺畅，长结果不卡顿
