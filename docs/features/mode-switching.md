# Feature: 模式切换（Mode Switching）

## 一句话

普通模式 ↔ 工作模式 之间手动切换 —— 由用户触发，桌宠扑棱翅膀演一段过场。

## 触发方式

V1 支持手动切换。三种入口：

| 入口 | 普通 → 工作 | 工作 → 普通 |
|---|---|---|
| **快捷键** | 系统全局快捷键（设置里改，默认 ⌘⇧H） | 同 |
| **右键菜单** | 桌宠身上右键 → "进入工作模式" | 桌宠头上右键 → "退出工作模式" |
| **拖拽** | 把桌宠拖到屏幕右上角自动钻入（**待 confirm**） | 工作模式下点头部拖出来到屏幕中央（待 confirm） |

> V1 不做自动切换（不基于打开 IDE 等行为推断）。后续 V2+ 可以加。

## 切换动画

### 普通 → 工作

1. 桌宠当前位置 → 翅膀大力扑棱两下（夸张一点）
2. 整体朝右上角斜飞，路径是一段轻微弧线
3. 飞到屏幕右上角时，做一个"侧身钻入"的旋转
4. 收尾：变成藏起来的姿态（半头 + 单翅露出，蓝翅亮起）
5. 总耗时 ~600ms，缓动 ease-in-out

### 工作 → 普通

1. 露出来的翅膀先扑棱一下
2. 桌宠从屏幕右上角"走出来"（向左下方平移 + 翅膀挥动）
3. 出现在屏幕右下偏中位置（或回到上次普通模式时的位置 —— **待 confirm**）
4. 收尾：站定，左翅膀亮金色，恢复 base loop（呼吸 + idle）
5. 总耗时 ~600ms

## 模式指示

唯一的指示是头盔翅膀颜色：

| 模式 | 左翅膀 | 右翅膀 |
|---|---|---|
| 普通 | 金色亮 + 辐射线 | 灰 |
| 工作 | 灰 | 蓝色亮 + 辐射线 |

详见 [UI-UX-Style.md](../UI-UX-Style.md) 第 1 节。

## 设计取舍：工作模式下露出的是"哪一侧"翅膀？

我们定义"右翅膀亮 = 工作模式"。但工作模式下桌宠藏在屏幕右上角，按朴素的"半边身子在屏幕外"理解，露出来的应该是**左半边身体 + 左翅膀** —— 那玩家根本看不到右翅膀亮，模式指示就失效了。

**解决方案（V1 拟定）：**

桌宠是**侧身钻进去**的 —— 它面朝左（朝向气泡和工作区域），右半边背对屏幕外。所以露出来的就是它身体的右半边 + 右翅膀。这一只翅膀亮蓝色就是工作模式指示。

**叙事上**：桌宠像是"扒着屏幕边缘往里看"，符合"我在工位上盯着工作"的观感。

**实现上**：工作模式的 SVG 不需要镜像 —— 直接画一个面朝左的桌宠，head 中心在屏幕右边缘，左半部分被裁掉。

## 状态机（横跨两模式）

```
[normal_idle] --切换触发--> [transition_to_work] --(600ms)--> [work_hidden]
[work_hidden] --切换触发--> [transition_to_normal] --(600ms)--> [normal_idle]
```

切换过程中所有交互（hover、拖放、点击）都被忽略。

## 验收标准

- 切换动画总时长 ≤ 800ms
- 切换中桌宠不"跳变" —— 必须是连续的位移和旋转
- 翅膀颜色变化在动画的最后 100ms 内完成（避免"还在飞行就变色"）
- 快捷键无论桌宠当前是什么状态都生效（除睡眠 zzz 时 —— 此时桌宠先醒，再切换）

## 未决问题

- 拖拽切换（拖到右上角自动进入）是否做？体感很好，但需要做"接近右上角"的命中检测
- 工作 → 普通 时，桌宠落在哪个位置？记住上次位置 / 总是屏幕中央 / 跟随当前鼠标
- 切换中是否允许打断（用户连续按两次快捷键）
