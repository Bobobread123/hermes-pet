# Feature: 设置面板（Settings）

## 一句话

桌宠的所有配置项都在这里 —— 入口少，所以这一面要干净、好懂、不啰嗦。

## 入口

- 桌宠右键菜单 → 设置
- 系统托盘图标（如果有） → 设置
- 设置面板**也是退出桌宠的唯一入口**（顶部右上角的 × 关掉的是设置窗口；退出桌宠在面板内有专门按钮）

## 面板结构

设置面板是一个独立窗口（约 480 × 600px），分几个区块从上到下：

### 1. 通用

- 开机自启（开关，默认开）
- 模式切换快捷键（默认 `⌘⇧H`，可重新录入）
- 桌宠默认起始位置：屏幕中央 / 右下角 / 上次位置（单选）

### 2. 后端 —— Hermes Agent

> **V1 默认走自动发现**（启动时按顺序探测 `$HERMES_BIN` → `which hermes` → `~/.local/bin/hermes` / Homebrew 路径等）。下面这些项是 fallback / 高级用户用，详见 [../tech.md](../tech.md) §4。

- **检测到的 Hermes 路径**（只读显示，灰色字）+ "覆盖" 按钮 → 改为手动指定
- Hermes CLI 路径（手动覆盖时才填，默认空）
- 工作目录（CLI 在哪个 cwd 下运行；默认 = 桌宠 .app 所在目录或用户主目录）
- 环境变量（多行 K=V，附加给子进程）
- 高级开关：
  - `--ignore-rules`（默认关）—— 让桌宠跑出来的会话不读 AGENTS.md / SOUL.md
  - `--ignore-user-config`（默认关）—— 不读 `~/.hermes/config.yaml`
  - 预加载 skills 列表（逗号分隔，传给 `-s`）
- "测试连接" 按钮 → 跑 `hermes chat -Q --source tool --accept-hooks -q "ping, reply with 'pong' only"`，回显是否成功

### 3. System Prompt 默认值

每个气泡一个独立的多行编辑器：

| 气泡 | 默认 prompt 编辑 |
|---|---|
| research | 万能分析骨架（详见 research-bubble.md） |
| 对话 | 默认对话人格（用户自填） |
| cowork | 任务执行型（详见 cowork-bubble.md） |

每个区块底部有：保存 / 重置为内置默认。

> 对话气泡里"临时改 prompt"修改的是会话内的实例，不会改这里的默认值（除非用户显式点对话浮窗里的"保存为默认"）。

### 4. 互动 / 陪伴

- 声音（开关，默认开）—— 关掉则摸头哼唧、睡眠呼吸、完成"叮"全部静音
- 进入睡眠的鼠标空闲阈值（滑杆，1-10 分钟，默认 3）
- idle 动画频率（滑杆，30s-180s，默认 60s）
- **亲密度查看**：显示 `pat_count`、`total_active_seconds` 等数字 + "重置亲密度"按钮（带二次确认）

### 5. 拖放

- 接受的最大单文件大小（默认 100MB）
- 普通模式拖入内容的处理策略（A/B/C 三选一，详见 drag-and-drop.md 的"未决问题"）

### 6. 关于

- 版本号
- 一个 GitHub 链接（如果未来有）
- 一个反馈入口（mailto: 或链接）

### 7. 退出（独立区块、底部）

- 一个红色或强调色按钮："退出 Hermes 桌宠"
- 二次确认弹窗："桌宠会消失，再次启动可在 ... 找到"
- 确认后桌宠进程结束

## 设置存储

- 文件位置：`~/Library/Application Support/HermesPet/settings.json`
- 启动时读，修改时即时写（不需要"保存"按钮）
- prompt 默认值的"保存"按钮例外 —— 因为内容长且改完想要明确确认
- 亲密度数据存独立文件 `affection.json`（详见 normal-mode.md）

## 视觉

- 风格与桌宠一致：暖米调背景、圆角、minimal
- 标题字体 16px medium，正文 14px regular
- 区块之间 24px 间距，区块内项目 12px 间距
- 不用花里胡哨的图标，纯文字 + 必要的输入控件

## 验收标准

- 修改任意项目（除 prompt）→ 写入文件 < 100ms
- "测试连接"按钮真的拉起 CLI 子进程并返回结果
- 重置亲密度后立即生效，桌宠下次摸头从 1 开始计数
- 退出按钮 → 桌宠进程在 1s 内退出干净（不留僵尸子进程）

## 未决问题

- 是否需要"导入/导出设置"（用户在多设备共用）—— V1 不做
- 是否要为 system prompt 提供模板市场 / 社区分享 —— V1 不做
- 设置窗口本身需不需要也"萌系" —— 倾向于不要，让设置保持工具感，桌宠自己萌就够了
