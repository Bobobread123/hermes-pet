# Feature: cowork 气泡

## 一句话

工作模式下三个气泡的最下面一个 —— 把一个完整任务交给 Hermes，它干完了挥手叫你来看。

## 定位

cowork 是"派单"模式 —— 用户描述一个任务，Hermes 负责完成。**一次一个任务，做完为止**。  
和 research 的区别：research 是问问题给框架化答案；cowork 是干活给可交付物（一份文档、一段代码、一个表格、一段总结）。  
和 对话 的区别：对话是多轮聊天找方向；cowork 是一次性派单不来回拉扯。

## 输入

- placeholder：`交个任务…`
- 单行输入；回车提交
- 支持拖入：文件 / URL / 文本会作为任务的"附件 / 上下文"被附在请求里

## System Prompt（V1）

V1 不做复杂模板，给一个简单的任务执行型 prompt：

```
你是一个执行型助手。我会给你一个具体任务，请直接完成它，不要反问、不要罗列计划，
做完直接给我可交付物。如果任务确实需要澄清才能继续，先用一句话提一个最关键的澄清问题，
其余假设直接做。
```

> 这个 prompt 默认值可在设置里改。

## 后端调用

同 research —— 单轮 spawn，无上下文：

```bash
hermes chat -Q --accept-hooks \
  -q "<cowork_system_prompt>\n\n<user_input>"
```

cowork 任务通常会触发 Hermes 调工具（写文件、跑命令等），所以：
- **不开 `--yolo`**：让 Hermes 自己的危险命令策略生效（用户在 `~/.hermes/config.yaml` 里配的）
- `--accept-hooks` 还是要开，因为桌宠是无 TTY 环境
- `--source tool` 因 2026-04-29 实测性能异常，V1 暂不带

详见 [../tech.md](../tech.md) §3.1 / §3.2。

## 视觉反馈

- 输入提交 → 红脸（任务运行）
- cowork 任务通常时间更长，桌宠的红脸状态可能持续几分钟
- **任务运行期间，工作模式即使收起气泡（鼠标离开），桌宠头部还是红脸的** —— 这是 cowork 区别于其他气泡的视觉重点（让用户瞥一眼右上角就知道"还在干"）
- 流结束 → 红脸消退 + 挥手 + 气泡 / session tab 上小红点；用户打开气泡或切到对应 tab 后红点清除

## 结果展开形式

点击 cowork 气泡：

- 展开成一个浮窗（宽度与上方 cowork 输入 pill 保持一致，放可交付物）
- 内容：
  - Tab 区：当前气泡的 session tabs，可新建 / 切换 / 删除本地 cowork session
  - 顶部：原任务描述
  - 主体：Hermes 输出（Markdown 渲染，支持代码、表格、长列表）
  - 底部：复制按钮 + "导出为文件" 按钮（如果输出是文档/代码型，提供保存）+ "丢给对话气泡讨论"按钮
- Esc / 点击外部 → 收起，结果保留

## 多任务 / 串行 / 并发

- V1 同一时刻 cowork 气泡只允许一个任务在跑；提交时如果上一个还没结束 → 提示用户"上一个还没完，要排队还是取消"（**待 confirm**）
- 多个气泡之间允许并发（research + cowork 同时跑），桌宠红脸只要任一个在跑就保持
- V1 session tab 只管理桌宠本地展示记录；删除 tab 不删除 Hermes Agent 底层 session store

## 错误处理

- 子进程异常退出 → 同 research：困惑表情 + 错误信息 + 重试按钮
- cowork 类任务可能因为 Hermes 自身工具调用失败 → 错误内容由 stdout 末尾返回，UI 解析后展示

## 验收标准

- 提交任务到红脸出现 < 100ms
- 长任务（> 1 分钟）红脸状态稳定不丢失
- 收起气泡后红脸状态依然在头上保持，hover 重新打开仍能看到该气泡的红点
- 结果浮窗可正确导出文件（如果有）
