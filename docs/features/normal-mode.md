# Feature: 普通模式（Normal Mode）

## 一句话

桌宠"下班"状态 —— 全身露在桌面上，会动会反应，但不打扰。

## 视觉

- 桌宠全身可见（头 + 身 + 双臂 + 双脚）
- 头盔**左翅膀亮金色（#FFB840）+ 三道辐射状细线**，右翅膀灰色
- 默认表情：眼睛圆睁，嘴角微弯
- 可被鼠标拖动到桌面任意位置（V1 暂不规定停靠规则）

## 行为

### B1. 微呼吸（base loop）

身体上下浮动 ±1px，周期约 2.5s，让桌宠看起来"活着"。任何时候都开着，除非进入 zzz 睡眠。

### B2. Idle 动画

每 60 秒触发一次随机 idle，二选一：
- **转圈**：身体原地旋转 360°，0.8s
- **扭头**：头部左右各转一次，0.6s

不走动 —— 桌宠不会自己在屏幕上挪位置（V1 范围）。

### B3. 摸头

- 触发：单击桌宠头部（不是身体或翅膀）
- 反馈动画：眯眼 + 翅膀抖两下（500ms），可选小哼唧声
- 副作用：亲密度 +1（详见 affection 部分）

### B4. 睡眠 zzz

- 触发：鼠标连续 N 分钟未进入桌宠周围一圈区域（V1 拟定 N = 3min，可在设置改）
- 进入动画：眼睛渐变成两条闭眼弧线，头顶飘出 zzz（小字气泡）
- 持续：呼吸变缓（周期 4s），zzz 字符循环上飘
- 退出：鼠标靠近 / 点击它 → 立刻醒（眼睛恢复，眨一下）

### B5. 拖东西给它

- 把文件 / 选中文本 / URL 拖到桌宠身上时，它朝鼠标方向倾斜 + 张嘴
- 松开后嘴合上，微弹一下
- 普通模式下接住的内容默认进入"对话"输入（切换到工作模式时已经填好）—— **这一条待 confirm**

### B6. 模式切换出口

普通模式没有可见的 UI 按钮触发"切到工作模式"。建议入口：
- 全局快捷键（设置里可改）
- 右键菜单里的"进入工作模式"
- 把桌宠拖到屏幕右上角自动钻进去（待定，做了会很有故事感）

## 亲密度（Affection）

V1 阶段的亲密度是**默默累计、不展示**：

- 数据：`pat_count`（摸头次数）、`last_pat_at`（上次摸头时间）、`total_active_seconds`（桌宠总在线时间）
- 存储：本地文件（详见 tech.md）
- V1 不影响行为；V2 可能：
  - 高亲密度 → idle 时更倾向看向鼠标 / 用户那一侧
  - 低亲密度 → 更安静、动作更小

## 状态机

```
            点头部
[base] ---------> [head_pat] ---(500ms)---> [base]
   |
   | 60s
   v
[idle anim] ---(800ms)---> [base]
   |
   | mouse-away N min
   v
[sleep zzz] ---mouse-near---> [wake] ---> [base]
   |
   | drag file in
   v
[catch] ---(500ms)---> [base]
```

## 与其他特性的关系

- 切到工作模式 → 跳到 [work-mode.md](work-mode.md) 的入口动画
- 摸头次数累积写入本地（settings.md 的"亲密度查看/重置"读这份数据）
- 拖放细节走 [drag-and-drop.md](drag-and-drop.md)

## 验收标准

- 桌宠常驻桌面，启动到出现 < 3s
- idle 每分钟一次，连续观察 5 分钟应至少看到 4 次（允许 1 次 jitter）
- 摸头点击响应延迟 < 100ms
- 鼠标 3min 不靠近 → zzz 出现
- 鼠标进入桌宠周围 50px 圆 → 立即醒来
